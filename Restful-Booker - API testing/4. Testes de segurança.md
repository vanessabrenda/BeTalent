## Create Booking(reserva) sem token

### Apesar do site restful indicar que o create pode sem feito sem token, não é o recomendado para a segurança, sendo assim, requer análise para possibilidade de alteração e restringir para uso obrigatório de token.

## Update de booking(reserva) sem token

### O site restful indica que é opcional o uso de token para realizar update parcial ou update completo, mas as próprias APIs não permitem as alterações sem token, o mais indicado é preservar a necessidade de uso de Token e alterar a documentação do site.
