# Plano de Testes - Sauce Demo
## Objetivo
  Esse documento tem por objetivo descrever quais estratégias foram usadas para validar os cenários de testes da plataforma de e-commerce Sauce Demo, garantindo a qualidade do sistema.

## Escopo de testes
### Tipos de testes utilizados
  1. Funcionais
  2. Responsividade
  3. Acessibilidade
  4. Automatizados
###  Estratégia de Testes
  **Testes funcionais:** Linguagem BDD   
  **Testes de responsividade:** DevTools do Chrome   
  **Testes de acessibilidade:** NVDA   
  **Testes automatizados:** VSCode, Playwright, Chrome, Node.js   
  **Ambiente de homologação:** https://www.saucedemo.com/

## Critérios de aceite
  Os cenários da seção "Cenários críticos" precisam funcionar em 100% dos casos. O restante dos casos poderão apresentar inconsistências, mas não bloqueará o uso do sistema.
